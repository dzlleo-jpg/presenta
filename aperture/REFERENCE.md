# DNA Forge · Reference Library

> 给 dna-forge 在 Phase 1(Identity & References)和 Phase 7(Signatures)使用的灵感库。
> 不是"风格大全",而是**与 Aperture 同纲领的 8 个高质量参考** + 融合范式 + signature 分类法。

`v0.1 · 2026-04`

---

## 0. 用法

这个文件**不在每次 dna-forge 调用时都读** — 只在以下时机读:

- **Phase 1 Identity**:用户描述参考但说不清,或想"找个类似 X 但又不是 X"的风格 → 读 §1
- **Phase 1 末尾的融合决策**:用户给了多个参考但比例没定 → 读 §2 融合范式
- **Phase 7 Signatures**:用户做完 active signatures,需要思考"还可以加什么 experimental" → 读 §3 signature 分类法

不要把这里的 8 个风格照抄进新 DNA。**它们是参考点,不是模板**。每个新 DNA 都应该有自己独特的融合和 signature tension。

## 1. 8 个精选风格参考

每个条目格式:**血统 · 精神 · Style Locked 关键 · 适合融合什么 · 不适合什么**。

### 信息建筑派 — "数据不是装饰,是建筑材料"

#### 1.1 Pentagram

- **血统**:多伦多 1972 年成立的设计公司,出过 Paula Scher、Michael Bierut、Eddie Opara 等大师
- **精神**:**字体即主角**。颜色极度克制(黑白 + 1 色),版式靠字号 / 字重 / 字距三轴拉开层级
- **Style Locked 关键**:大胆的字号比(常 8:1 ~ 12:1);单一 accent 色,常是品牌色;留白比例 60%+;敢用单字 / 标点作为构图主体
- **适合融合什么**:任何需要"editorial 高级感"的项目;Aperture 已经吸收了这一脉的"克制 + 字号比"
- **不适合什么**:复杂数据可视化(Pentagram 喜欢用文字代替图表);消费类温暖品牌
- **代表作**:NYT Magazine 改版、AIGA、Vanity Fair 重塑

#### 1.2 Müller-Brockmann(瑞士国际主义)

- **血统**:Josef Müller-Brockmann 1950s,瑞士设计学派核心
- **精神**:**8pt 数学基线网格 + 绝对对齐**。功能主义至上,装饰即罪恶
- **Style Locked 关键**:严格栅格;最多 2 种字号;Helvetica / Akzidenz Grotesk 类无衬线;留白通过对齐而非空旷
- **适合融合什么**:数据报告、学术演讲、瑞士银行等需要"被认真对待"的品牌;Aperture 的 §3 Universal Floor 和 Bento gap 8px 都源自这里
- **不适合什么**:文学品牌、实验艺术、品牌叙事
- **代表作**:Beethoven 海报系列、苏黎世音乐厅 visual identity

#### 1.3 Fathom(信息可视化)

- **血统**:Ben Fry 在 MIT Media Lab 发起,延续到 Casey Reas / Processing 阵营
- **精神**:**科学期刊严谨 + 设计优雅**。精确数据可视化,注释设计化
- **Style Locked 关键**:冷灰 + 海军蓝主调;细线 hairline 大量使用;数字必用等宽字体;图例位置严格
- **适合融合什么**:投资 pitch、科研报告、需要数据驱动叙事的产品(Aperture × Fathom 适合做 SaaS dashboard)
- **不适合什么**:消费品、艺术、低密度内容
- **代表作**:Visualizing Data 一书;The New York Times 数据新闻团队风格

### 极简主义派 — "删减到无法再删"

#### 1.4 Build(Michael C. Place)

- **血统**:Build Studio,英国,曾负责 Wallpaper* / Penguin 等 editorial 品牌
- **精神**:**70%+ 留白 + 微妙字重对比 + 单 accent 战略运用 + 黄金比例**
- **Style Locked 关键**:字重对极端化(200 vs 600,中间禁用);accent 色每页只用 1 次,且位置语义化;栅格基于黄金比例 1.618
- **适合融合什么**:奢侈品牌、高端产品、premium SaaS;Aperture 的字重对和 accent 数量上限直接来自 Build
- **不适合什么**:数据密集 dashboard、消费类大众产品
- **代表作**:Wallpaper* magazine layouts、Penguin 系列书籍封面

#### 1.5 Kenya Hara(原研哉 / 日本设计中心 NDC)

- **血统**:无印良品 art director、武藏野美术大学;日本设计中心(NDC)社长
- **精神**:**間 (Ma) — 留白即内容,沉默即信息**。80%+ 极致留白;白色的层次(暖白 / 冷白 / 米白);减法设计
- **Style Locked 关键**:WaShi 和纸感底色;墨黑 + 朱红双色;字重永远轻(100-200);允许 NDC pictogram(几何 fill)
- **适合融合什么**:文化机构、奢侈品东亚市场、需要"安静的高级"的项目;Aperture 的 Japanese Editorial 子调整套来自这里
- **不适合什么**:B2B SaaS、数据 dashboard、技术产品
- **代表作**:Muji 整套品牌;白(《白》一书);Experience Japan 国家形象设计

#### 1.6 Experimental Jetset

- **血统**:阿姆斯特丹 1997 年三人小组(Marieke Stolk, Erwin Brinkers, Danny van den Dungen)
- **精神**:**单一视觉隐喻贯穿全场**。蒙德里安色系、字体即图形、反商业诚实
- **Style Locked 关键**:Helvetica + 强对比色块(红黄蓝);常用全大写无字距;每个项目一个 strong concept,执行到极端
- **适合融合什么**:艺术机构、概念展览、独立媒体;Aperture 的 Color Block 子调神似这种处理
- **不适合什么**:商业产品、需要"友好亲和"的品牌
- **代表作**:Whitney Museum 一系列展览设计、Stedelijk Museum

### 东方哲学派 — "留白 + 文学"

#### 1.7 Takram(日本 + 伦敦)

- **血统**:东京 + 伦敦双总部,设计与工程并重
- **精神**:**日式思辨 + 战略咨询气质**。概念原型优雅,图表即艺术,谦逊的精致
- **Style Locked 关键**:中性自然色(米白 / 灰 / 木色);严密的信息层级;插图风格抽象但有温度;不多用 accent
- **适合融合什么**:战略咨询、研究展示、东西方桥接型品牌
- **不适合什么**:大众消费产品、高密度数据 SaaS
- **代表作**:Suntory 品牌战略、KDDI 设计创新部

#### 1.8 Sagmeister & Walsh(纽约)

- **血统**:Stefan Sagmeister + Jessica Walsh,纽约
- **精神**:**意外色彩爆发 + 手工感融入数字**。正能量,实验排版但可读
- **Style Locked 关键**:大胆饱和色(常一页一种);手写字体 + 严肃无衬线;摄影 + 印刷质感叠加;每个项目都有一个"意外"瞬间
- **适合融合什么**:创意行业、艺术机构、想"被记住"的项目;Aperture 的 §12 signatures 哲学借鉴了这个 — "刻意的意外"
- **不适合什么**:严肃 B2B、政府 / 法律
- **代表作**:The Happy Show、HBO Max rebrand

## 2. 融合范式 Fusion Patterns

### 2.1 70/30(Aperture 用的)

**最常用、最稳**。一个主参考定 70% 的视觉语言,一个辅参考贡献 30% 的"补全部分"。

- 主负责:字体 / 色板 / 整体气质 / 留白哲学
- 辅负责:Aperture 主参考没解决的痛点(如 Polestar 太空,Apple Bento 补信息密度)
- **关键**:辅 30% 必须解决主 70% 的明显短板,而不是冲突

适合:有清晰立场的项目,不想"什么都是"的折中

### 2.2 60/30/10

**三参考,主 + 中 + 调味料**。10% 是"signature flavor",出现频率低但很显眼。

- 主 60%:气质底色
- 中 30%:能力补全
- 调味料 10%:让人一眼记住的"奇怪"细节(eg Aperture 的 NDC pictogram 在 Japanese Editorial 子调中)

适合:复杂多样化项目,需要 versatility

### 2.3 50/50 平衡

**风险高,慎用**。两个参考各占 50% 的"对位融合",最容易变成"四不像"。

只有当两个参考有**明确的互补轴**时才行得通(eg "瑞士网格的严谨" × "日式留白的诗意" — 一个负责空间结构,一个负责时间节奏)。

不适合:大多数情况;新 DNA 第一次造时强烈不建议

### 2.4 90/10 致敬式

**用 90% 一个参考,10% 是 strategic departure**。10% 是"我和这个参考的不同点"。

适合:已有强参考但需要一点差异化(eg 想做"Polestar 但更暖一点")

### 2.5 融合的反模式

避免:

- "所有参考各占 1/N" — 没有 commitment,平均化为 generic minimal
- "参考太多"(≥ 4 个) — 没人能记住 4 个参考的指纹,DNA 会失焦
- "参考之间没有互补轴" — 只是"我喜欢这两个" → 强行融合会撕裂

## 3. Signature 分类法

§12 Aperture Signature Library 有 5 active + 5 experimental。这一节把它们抽象成**机制类别**,方便造新 DNA 时设计 signature。

### 3.1 Scale Inversion 尺度反转

把"该大的"做小,"该小的"做大。
- Aperture #1 Punctuation as Subject(标点 → hero)
- Aperture #7 Caption as Subject(caption → hero,hero → caption)
- Aperture #10 The Punch(西文巨大,中文极小)

**机制核心**:违反"信息层级 = 视觉层级"的常识,但要有内容理由(self-aware)。

### 3.2 Element Promotion 元素晋升

把通常作为辅助的元素提升到主体地位。
- Aperture #6 Negative Space Logo(section marker → 整版构图)
- Aperture #2 Single Glyph Heroic(单字符占满画面)

**机制核心**:让"次要元素"承担"主要叙事"。

### 3.3 Spatial Intrusion 空间侵入

让内容侵入"不该去"的区域。
- Aperture #3 Edge-bleed Number(数字超出容器边)
- 类似手法:文字跨页、图像出血到导航条

**机制核心**:挑战"内容必须在容器内"的潜规则。

### 3.4 Reading Direction 阅读方向

打破默认的左到右、上到下阅读。
- Aperture #4 Vertical Tower(纵向塔式)
- 类似手法:斜排、镜像、回文

**机制核心**:CJK 友好度极高(Latin 难做)。

### 3.5 Type as Object 字符即物体

把字符当作几何对象处理,而不是阅读对象。
- Aperture #9 Type as Image(字母分散为构图列)
- 类似手法:字符堆叠、字符路径、字符变形

**机制核心**:适合品牌名 / 产品名展开页;字符数 4-7 是甜区。

### 3.6 Calculated Wrongness 蓄意错位

刻意打破对齐 / 节奏 / 颜色的预期。
- Aperture #8 The Misalignment(基线偏移 22px)
- Aperture #5 Asymmetric Number Pair(字号 / 基线刻意不齐)

**机制核心**:风险最高 — 不打配合就是"不细致"。必须有内容戏剧。

### 3.7 Punctuation as Statement 标点即声明

把标点(. , 「 」 — 等)处理成首要视觉元素。
- Aperture #1 Punctuation as Subject(标点放大到 ≥ 400px)

**机制核心**:Polestar 海报常用;CJK 中"。"和"、"特别有力。

### 3.8 造新 DNA 时如何用这个分类

每个新 DNA 至少 active 5-7 个 signatures,建议:
- **必有 1 个 Scale Inversion**(造冲击力)
- **必有 1 个 Reading Direction 或 Spatial Intrusion**(造记忆点)
- **必有 1 个 Element Promotion**(造主从颠倒戏)
- **可选 1-2 个 Calculated Wrongness 或 Punctuation as Statement**(高风险高回报)

不要 8 个 signatures 全是同一类(eg 全是 Scale Inversion),否则一份 deck 会感觉"老在重复同一个动作"。

## 5. 字体白名单(Aperture v0.4 §4.4 的扩展)

- **Geist** / **Geist Mono** — Vercel 出品,几何感克制,AI 味为零
- **Söhne** — Klim 出品,商业字体,Polestar / Apple 同源审美
- **GT America** — Grilli Type,Apple 风

### 5.2 高端 / 奢侈
- **Cabinet Grotesk** — Indian Type Foundry,免费可用
- **Satoshi** — Indian Type Foundry,premium 感强
- **Roobert** — 适合 SaaS 高端

### 5.3 Editorial / 文学(衬线)
- **Lyon Text** — Commercial Type,正式 editorial
- **Newsreader** — Production Type,可读性强
- **Instrument Serif** — 免费,带个性

### 5.4 数据 / 等宽
- **Geist Mono** — 等宽配 Geist 系
- **JetBrains Mono** — 程序员熟悉,数据可视化适用
- **IBM Plex Mono** — 免费,严肃感

### 5.5 CJK
- **PingFang SC** — macOS / iOS 默认
- **Source Han Sans CN / 思源黑体** — 开源跨平台
- **Yu Gothic / Hiragino** — 日文优选

### 5.6 黑名单(永远不主用)

见 Aperture v0.4 §4.4。不重复。

## 7. 这个文件本身的演化

REFERENCE.md 是 dna-forge skill 的"灵感库",会随着造的 DNA 越多,提取的范式越多,逐渐演化:

- v0.1(本文件)— 8 个高质量参考 + 5 个融合范式 + 7 个 signature 类别
- v0.2 — 加入用户造出来的 DNA 的"反向参考"(eg 如果造了 Boreal,可以总结它的 signature 进入分类法)
- v0.3 — 增加 production 案例(真实项目使用 DNA 后的反馈)

请保持精炼。"风格大全"是反 dna-forge 的 — 这个 skill 的价值是**判断力**,不是"什么都收"。
